# Contributing

Bug reports, device compatibility results, documentation corrections, and
focused code reviews are welcome.

Before proposing a code change:

1. Keep PMDO runtime data, Pokemon assets, Quest data, saves, APKs, signing
   keys, and local SDK files out of Git.
2. Keep changes small enough to review and explain the platform impact.
3. Add or update tests where practical.
4. Run the portable test suite and relevant APK audit before submitting.
5. Preserve all upstream and third party license notices.

Use GitHub Issues for ordinary bugs and feature discussions. Use the private
process in `SECURITY.md` for security vulnerabilities.

## AI transparency

The initial port was produced with extensive use of OpenAI Codex. Contributions
may use AI assisted tools, but contributors remain responsible for reviewing,
testing, licensing, and accurately describing everything they submit.
